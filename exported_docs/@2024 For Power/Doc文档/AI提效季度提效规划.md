2. 六月份规划

目标: 搭建质量检测平台，基于知识库和 Spoon 代码分析工具进行结合，单元化检测代码质量潜在 bug，可输出建设性的优化建议和具体代码，并对代码漏洞级别进行分类，减少测试人员手动发现 bug 的效率。

实现路径: Spring AI + 本地知识库+ 阿里系大模型(qwen) + Spoon 代码分析工具+ 私有化部署

工具展望: 按照自定义规则（定时 + 代码变动）进行输出代码质量检测报告  


3. 七月份规划

目标: 搭建半自动化需求到代码的工作流生成平台。

实现路径: 搭建代码半自动化工作流平台（dify 二次开发），借助知识库进行简单的需求分析 + 模板化代码生成 + 单元测试用例生成。

实现路径: Spring AI + 本地知识库+ 阿里系大模型(qwen) +Dify+ 私有化部署  
工具展望:  基于需求信息输出符合现有代码库结构的基础代码信息，提高编码效率
