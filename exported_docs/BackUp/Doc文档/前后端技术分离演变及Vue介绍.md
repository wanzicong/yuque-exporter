+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">了解前后端分离的变化，探究其历史演变</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">熟悉前端领域的关键概念和基本技术</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">掌握前端工程的开发步骤和流程</font>

## <font style="background-color:rgb(247, 247, 248);">1. 前后端演变历史</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">让我们穿越时光隧道，回溯技术的演进历程</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">后端技术：Spring, Spring MVC, MyBatis, Hibernate, iBatis</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">一度没有分离的岁月：JSP、各类HTML模板、Bootstrap，以及HTML、CSS、JavaScript的综合应用</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">基本原理：Java通过后端请求，经后端框架解析，渲染后端数据，生成HTML字符串，然后通过Response交付给浏览器，由Chrome V8引擎和Node.js完成渲染。</font>

## <font style="background-color:rgb(247, 247, 248);">2. 前后端不分离</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">早期，前后端紧密相连</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">后端承担一切，前端只是展示的工具</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">原则：Java解决一切</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">后端扛大旗，前端悠闲</font>

## <font style="background-color:rgb(247, 247, 248);">3. 前后端分离</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">为什么要分离？分离后有何利弊？</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">前后端分离的必要性和优劣势</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">前端的角色：数据渲染和用户交互，成为互联网的大门守护者</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">后端的角色：通过接口提供数据、处理参数、逻辑处理，静默英雄</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">分离的好处：明确的角色划分，并行开发，提高效率；UI交互设计师轻松加入开发；每个团队成员专注自己的任务</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">分离的弊端：开发人员成本增加，系统维护复杂化，从一个Java开发者变成前后端都懂的全能型选手</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">曾经一个接口定义，直接自己上手，现在前后端分离，需要为可爱的前端写接口文档，多了些乐趣！</font>

## <font style="background-color:rgb(247, 247, 248);">4. 作为后端开发，有必要了解前端吗？</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">确实有必要！</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">为什么？因为可以卷！开发不是一场撕逼大战！</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">前端开发流程和基本概念掌握后，能够定义更高扩展性的接口</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">当前端牛逼到不好说话时，你可以以专业术语和逻辑与之争辩</font>

## <font style="background-color:rgb(247, 247, 248);">5. 前端与后端的恩怨</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">明确角色定位，但业务中还有交集：</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">前端参数处理、拦截、是否可前端处理？</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">返回结果处理、字段映射、格式转化</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">跨域问题：前端说后端处理，后端说前端处理，到底由谁来？</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">跨域是前端还是后端的概念？</font>
+ <font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">总之：开发是一家，别互相伤害！</font>

## <font style="background-color:rgb(247, 247, 248);">6. 前端的基本概念和组成</font>
