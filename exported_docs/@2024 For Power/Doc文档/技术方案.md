## <font style="color:rgb(6, 6, 7);">一、功能模块概述</font>
### <font style="color:rgb(6, 6, 7);">（一）小程序端</font>
+ **<font style="color:rgb(6, 6, 7);">商品订单列表</font>**
    - <font style="color:rgb(6, 6, 7);">显示订单信息，包括订单状态、金额等。</font>
    - <font style="color:rgb(6, 6, 7);">根据订单状态和金额等条件，控制是否显示退款按钮。</font>
+ **<font style="color:rgb(6, 6, 7);">申请退款页面</font>**
    - <font style="color:rgb(6, 6, 7);">提供申请金额、申请说明、联系电话等字段填写。</font>
    - <font style="color:rgb(6, 6, 7);">校验申请条件，如订单时间限制、金额限制等。</font>
    - <font style="color:rgb(6, 6, 7);">提交退款申请后，等待商家受理。</font>
+ **<font style="color:rgb(6, 6, 7);">退款详情页面</font>**
    - <font style="color:rgb(6, 6, 7);">显示退款申请的详细信息，包括申请时间、状态等。</font>
    - <font style="color:rgb(6, 6, 7);">提供撤销退款申请的操作。</font>

### <font style="color:rgb(6, 6, 7);">（二）后台端</font>
+ **<font style="color:rgb(6, 6, 7);">服务订单列表</font>**
    - <font style="color:rgb(6, 6, 7);">增加状态筛选功能，支持筛选子状态。</font>
    - <font style="color:rgb(6, 6, 7);">显示订单信息，根据订单状态和金额等条件，控制是否显示退款按钮。</font>
+ **<font style="color:rgb(6, 6, 7);">退款管理页面</font>**
    - <font style="color:rgb(6, 6, 7);">提供退款申请的列表展示，包括申请时间、金额、状态等信息。</font>
    - <font style="color:rgb(6, 6, 7);">支持受理退款申请，包括同意受理、同意受理并发起退款等操作。</font>
    - <font style="color:rgb(6, 6, 7);">提供撤销退款申请的操作。</font>
+ **<font style="color:rgb(6, 6, 7);">财务退款申请页面</font>**
    - <font style="color:rgb(6, 6, 7);">提供退款申请的填写和提交功能，包括金额、退款明细、交易详情等字段。</font>
    - <font style="color:rgb(6, 6, 7);">根据退款状态，更新订单的退款状态。</font>
+ **<font style="color:rgb(6, 6, 7);">后台退款 Tab 页面</font>**
    - <font style="color:rgb(6, 6, 7);">在服务订单中增加退款 Tab 页面，展示订单的退款信息。</font>
    - <font style="color:rgb(6, 6, 7);">提供退款状态筛选功能，包括待受理、受理通过、退款中、退款成功、已拒绝、已关闭等状态。</font>
    - <font style="color:rgb(6, 6, 7);">显示退款信息的详细字段，如申请退款金额、申请人、发起退款时间等。</font>

## <font style="color:rgb(6, 6, 7);">二、配置项</font>
### <font style="color:rgb(6, 6, 7);">（一）服务模板页面</font>
+ **<font style="color:rgb(6, 6, 7);">售后方式配置</font>**
    - <font style="color:rgb(6, 6, 7);">线上退款时间限制：默认值为15分钟，正整数。</font>
    - <font style="color:rgb(6, 6, 7);">是否自动受理退款：默认值为否。</font>
    - <font style="color:rgb(6, 6, 7);">是否自动发起退款申请：当自动受理退款选择是时，出现该选项，默认值为否。</font>
    - <font style="color:rgb(6, 6, 7);">是否发起自动退订申请：默认值为是。</font>

### <font style="color:rgb(6, 6, 7);">（二）财务缴费配置</font>
+ <font style="color:rgb(6, 6, 7);">在线退款默认经办人：未配置时，系统自动处理。</font>

## <font style="color:rgb(6, 6, 7);">三、逻辑要求</font>
### <font style="color:rgb(6, 6, 7);">（一）退款按钮显示逻辑</font>
+ **<font style="color:rgb(6, 6, 7);">不显示退款按钮的情况</font>**
    - <font style="color:rgb(6, 6, 7);">订单状态非已支付。</font>
    - <font style="color:rgb(6, 6, 7);">原始订单金额为0。</font>
+ **<font style="color:rgb(6, 6, 7);">显示退款按钮的情况</font>**
    - <font style="color:rgb(6, 6, 7);">订单状态为已支付。</font>
    - <font style="color:rgb(6, 6, 7);">原始订单金额大于0。</font>
    - <font style="color:rgb(6, 6, 7);">从未申请过退款。</font>

### <font style="color:rgb(6, 6, 7);">（二）申请退款校验逻辑</font>
+ <font style="color:rgb(6, 6, 7);">订单时间限制：基于配置的15分钟默认值。</font>
+ <font style="color:rgb(6, 6, 7);">可申请金额为订单金额和账单可退金额的最小值。</font>
+ <font style="color:rgb(6, 6, 7);">申请金额默认为订单已支付金额，不可修改。</font>
+ <font style="color:rgb(6, 6, 7);">申请说明限制为200字，同步到退款申请页的说明。</font>
+ <font style="color:rgb(6, 6, 7);">联系电话默认为当前登录电话，可修改。</font>

### <font style="color:rgb(6, 6, 7);">（三）受理逻辑</font>
+ <font style="color:rgb(6, 6, 7);">订单提醒：提交成功后，根据配置，若需人工受理，则消息提醒商家管理员；若自动受理，则无需消息提醒。</font>
+ <font style="color:rgb(6, 6, 7);">受理成功或失败：包含失败原因，退款结束。</font>

### <font style="color:rgb(6, 6, 7);">（四）退款成功逻辑</font>
+ <font style="color:rgb(6, 6, 7);">审核通过后，自动触发商户号退款，原路返回到客户账户。</font>
+ <font style="color:rgb(6, 6, 7);">如果是后台退款，时间取退款的操作时间。</font>
+ <font style="color:rgb(6, 6, 7);">账单的原始应收金额变为0。</font>
+ <font style="color:rgb(6, 6, 7);">优惠券退回给客户，状态更新为待使用或已过期。</font>
+ <font style="color:rgb(6, 6, 7);">小程序个人优惠券和企业优惠券变为待使用状态。</font>
+ <font style="color:rgb(6, 6, 7);">账单其他的减免或调整或坏账明细作废。</font>
+ <font style="color:rgb(6, 6, 7);">实际应收金额变为0。</font>

### <font style="color:rgb(6, 6, 7);">（五）撤销退款逻辑</font>
+ <font style="color:rgb(6, 6, 7);">如果在小程序撤销退款申请的同时，后台也在操作撤销，则提示“退款申请已撤销，请勿重复操作”。</font>
+ <font style="color:rgb(6, 6, 7);">撤销成功后，系统提示“撤销退款成功”。</font>
+ <font style="color:rgb(6, 6, 7);">如果在订单的待退款状态确认撤销后，后台订单的退款单状态会变为“已关闭”。</font>

### <font style="color:rgb(6, 6, 7);">（六）财务退款逻辑</font>
+ <font style="color:rgb(6, 6, 7);">从财务模块退款的，不原路返回，人工手动退订。</font>
+ <font style="color:rgb(6, 6, 7);">财务退款成功，订单的退款状态更新为已退款。</font>
+ <font style="color:rgb(6, 6, 7);">财务的退款明细作废后，财务的退款状态更新为待退款，订单退款状态更新为退款中。</font>

## <font style="color:rgb(6, 6, 7);">四、流程设计</font>
### <font style="color:rgb(6, 6, 7);">（一）小程序退款流程</font>
1. <font style="color:rgb(6, 6, 7);">商品订单列表</font>
2. <font style="color:rgb(6, 6, 7);">点击申请退款</font>
3. <font style="color:rgb(6, 6, 7);">填写信息（申请金额、申请说明、联系电话）</font>
4. <font style="color:rgb(6, 6, 7);">等待商家受理</font>
5. <font style="color:rgb(6, 6, 7);">查看退款详情</font>
6. <font style="color:rgb(6, 6, 7);">撤销退款申请</font>
7. <font style="color:rgb(6, 6, 7);">再次申请退款</font>

### <font style="color:rgb(6, 6, 7);">（二）后台退款流程</font>
1. <font style="color:rgb(6, 6, 7);">服务订单列表</font>
2. <font style="color:rgb(6, 6, 7);">点击申请退款</font>
3. <font style="color:rgb(6, 6, 7);">跳转到财务退款管理</font>
4. <font style="color:rgb(6, 6, 7);">填写退款申请（金额、退款明细、交易详情）</font>
5. <font style="color:rgb(6, 6, 7);">提交退款申请</font>
6. <font style="color:rgb(6, 6, 7);">受理退款申请</font>
7. <font style="color:rgb(6, 6, 7);">退款成功或失败</font>
8. <font style="color:rgb(6, 6, 7);">撤销退款申请</font>

## <font style="color:rgb(6, 6, 7);">五、其他注意事项</font>
### <font style="color:rgb(6, 6, 7);">（一）异常状态处理</font>
+ <font style="color:rgb(6, 6, 7);">商户号余额不足退款失败，展示原因提示。</font>
+ <font style="color:rgb(6, 6, 7);">对应操作为重新执行扣款、详情、退款。</font>
+ <font style="color:rgb(6, 6, 7);">点击重新执行扣款，则从商户号再次退款。</font>

### <font style="color:rgb(6, 6, 7);">（二）数据同步</font>
+ <font style="color:rgb(6, 6, 7);">新架构数据同步至退款单、退款明细。</font>
+ <font style="color:rgb(6, 6, 7);">旧架构保持现状。</font>

### <font style="color:rgb(6, 6, 7);">（三）状态同步</font>
+ <font style="color:rgb(6, 6, 7);">退款状态新增“异常”状态。</font>
+ <font style="color:rgb(6, 6, 7);">审核状态和退款状态的经办人配置影响范围包括退款单、退款明细、物业账单、资源账单、客户账单、账单台账实收数据的明细的操作人、收据列表开具人、退款人、退款明细表退款人等。</font>

### <font style="color:rgb(6, 6, 7);">（四）通知提醒</font>
+ <font style="color:rgb(6, 6, 7);">退款成功后，提醒用户和管理员（仅限于产业的账单提醒）。</font>
+ <font style="color:rgb(6, 6, 7);">退款失败时，仅产业的账单提醒管理员。</font>

## <font style="color:rgb(6, 6, 7);">六、涉及页面</font>
+ <font style="color:rgb(6, 6, 7);">小程序端：商品订单列表、申请退款页面、退款详情页面。</font>
+ <font style="color:rgb(6, 6, 7);">后台端：服务订单列表、退款管理页面、财务退款申请页面、后台退款 Tab 页面、账单缴费配置、核销流水页面。</font>

```sql
CREATE TABLE `refund_applications` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `order_id` VARCHAR(255) NOT NULL COMMENT '订单编号',
  `product_service_id` VARCHAR(255) NOT NULL COMMENT '商品/服务名称',
  `product_service_name` VARCHAR(255) NOT NULL COMMENT '商品/服务名称',
  `application_amount` DECIMAL(10, 2) NOT NULL COMMENT '申请金额',
  `application_reason` TEXT COMMENT '申请说明',
  `application_time` DATETIME NOT NULL COMMENT '申请时间',
  `refund_status` VARCHAR(50) NOT NULL COMMENT '退款状态',
  `merchant_processing_time` DATETIME COMMENT '商家受理时间',
  `refund_end_time` DATETIME COMMENT '退款结束时间',
  `refund_success_time` DATETIME COMMENT '退款成功时间',
  `refund_method` VARCHAR(50) COMMENT '退款方式',
  `remarks` TEXT COMMENT '备注'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
