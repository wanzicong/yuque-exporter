# 直播技术方案设计


```plsql
CREATE TABLE `live_wechat_room` (
  `id` bigint(20) unsigned NOT NULL AUTO_INCREMENT,
  `name` varchar(32) NOT NULL DEFAULT '0' COMMENT '直播间名称',
  `coverImg` varchar(32) NOT NULL COMMENT '直播间ID',
  `startTime` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '直播间开始时间',
  `endTime` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT  '直播计划结束时间',
  `anchorName` varchar(32) NOT NULL COMMENT '主播名',
  `roomid` varchar(32) NOT NULL COMMENT '直播间ID',
  `liveStatus` int(1)  NOT NULL COMMENT '直播间状态。101：直播中，102：未开始，103已结束，104禁播，105：暂停，106：异常，107：已过期',
  `shareImg` varchar(32) NOT NULL COMMENT '分享图，填入mediaID（mediaID获取后，三天内有效）',
  `liveType` int(1)  NOT NULL COMMENT '直播类型，1 推流 0 手机直播',
  `closeLike` int(1)  NOT NULL COMMENT '是否关闭点赞 【0：开启，1：关闭】（若关闭，观众端将隐藏点赞按钮，直播开始后不允许开启）',
  `closeGoods` int(1)  NOT NULL COMMENT '是否关闭货架 【0：开启，1：关闭】（若关闭，观众端将隐藏商品货架，直播开始后不允许开启）',
  `closeComment` int(1)  NOT NULL COMMENT '是否关闭评论 【0：开启，1：关闭】（若关闭，观众端将隐藏评论入口，直播开始后不允许开启）',
  `closeKf` int(1)  NOT NULL COMMENT '是否关闭客服 【0：开启，1：关闭】 默认关闭客服（直播开始后允许开启）',
  `closeReplay` int(1)  NOT NULL COMMENT '是否关闭回放 【0：开启，1：关闭】默认关闭回放（直播开始后允许开启）',
  `isFeedsPublic` int(1) NOT NULL COMMENT '是否开启官方收录，1 开启，0 关闭',
  `createrOpenid` varchar(32)  NOT NULL COMMENT '创建者openid',
  `feedsImg` varchar(32)  NOT NULL COMMENT '官方收录封面',
  `replayMediaUrl` varchar(32)  NOT NULL COMMENT '回放url',
  `replayCreateTime` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP  COMMENT '回放创建时间',
  `replayExpirTime` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP   NOT NULL COMMENT '回放过期时间',
  `goods` varchar(200)  NOT NULL  DEFAULT '' COMMENT '直播间商品',
  `gmt_create` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `gmt_modified` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
  `creator` varchar(32) NOT NULL DEFAULT 'sys' COMMENT '创建人',
  `modifier` varchar(32) NOT NULL DEFAULT 'sys' COMMENT '修改人',
  `is_deleted` tinyint(4) NOT NULL DEFAULT '0' COMMENT '是否删除：1(删除) 0(未删除)',
  PRIMARY KEY (`id`),
  KEY `idx_roomid` (`roomid`),
  KEY `idx_createrOpenid` (`createrOpenid`)
) ENGINE=InnoDB AUTO_INCREMENT=1028 DEFAULT CHARSET=utf8mb4 COMMENT='微信直播间表';
```

根据定时任务进行同步：

时间间隔 1 min

每次请求三方数据全量数据（当前数据量比较小目前采用此方案）

1. 数据在表中不存在：添加操作
2. 数据在表中存在：跟新操作，比如更新直播状态
3. 删除在表中存在但是三方接口中不存在的直播间



直播开播发送消息逻辑：

根据定时任务轮训表中的直播间

1. 根据当前时间-当前时间后三秒筛选已经开播的直播间
2. 找到对应设计师信息，同时获取全部的成衣和图稿，找到对应关注的主播信息
