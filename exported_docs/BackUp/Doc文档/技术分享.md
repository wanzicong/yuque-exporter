# 前后端分离
前端和后端进行分离， 为什么分离，分离后带来哪些好处和弊端，不分离行不行

前后端分离后前端承载什么角色？ 后端承载什么角色？

后端： 以接口形式 提供接收参数--》处理参数--》逻辑处理--》返回页面期待结果，（默默无闻）

前端：提供数据渲染能力，和用户交互逻辑，承载互联网的入口的角色（叼的很）

分离带来的好处是：开发的逻辑角色划分清晰，开发效率并行，提高。ui交互设计可以很快的参与其中，每个岗位只需关心自己的开发逻辑和任务即可

分离带来弊端：开发人员成本变高，系统维护成本变高，原来一个java开发就可以搞定，现在必须找一个后端开发和前端开发

本来一个定义的接口，、。自己肯定熟悉的很直接接入就完事了，但是前后端分离后，还得给那些小可爱们写接口文档，想想就开心！！

# 前端的基本概念以及组成
随着技术的演变和业务的复杂度以及市场需求不断的迭代，前端体系由前端三剑客（js ，html ，css）演变为了现在错综复杂的各种概念和框架，以及开发工具。

前端基本技术：

  1.  javascript：一门编程语言，解释型的语言，直接运行，无需编译，特点：随意， 承担角色：让页面动起来，页面dom元素处理。

2. html：一种结构化的文本语言，严格意义上不属于编程语言。承担角色：页面内容的全部载体，最常用的元素： div a input span ul li
3. css： 页面样式处理， 比如控制页面背景色，dom元素之间的关系，也就是页面的布局，承担角色：让页面变得更好看



后期随着技术的发展，前端体系变得更加的庞大：

各种ui框架：原生css 替代工具, bootstrap , elementui ,antdesign 。。

js语言运行环境和规范：node.js ，typescript

包管理工具：npm cnpm pnpm yarn

资源打包工具：webpack

前端工程化的开发框架： vue react ..... vue-cli ，vite



# 前端框架介绍
前端框架丰富多彩，但笔者觉得vue是最牛逼的，别问为什么，问就是只会vue....

所以关于前端的学习，我们从vue开始！！

vue是一个前端开发框架，类似于后端开发框架的springboot，vue是华人开发的一门不错的技术，为无数热爱前端开发的小可爱们指明了道路。文档对中文支持的非常好。

vue使用分为两个体系：一个是vue2，一个是vue3，目前最新的技术框架是vue3，从二到三是一个非常大的迭代，很多编程语法和习惯都变了很多，毫不夸张的说 vue2 和 vue3 不亚于 jdk 7 -》8 的变化

vue 是一个开发框架，最大的特点是响应式，所谓的响应式就是，数据的变化可以影响到页面的渲染，同时如果页面中的数据发生了变化可以影响到程序中的变量值。

# 前端框架基本组成
可以类比前端基本三剑客： javascript，css  ， html

最为一个工程化的框架肯定会有更多的功能：

前端ui组件：element ui， 一个可以和vue进行完美适配的前端ui，（毫不夸张的说前端开发 大部分时间都在从 ui组件的官网到自己的代码中，前端开发 = ui组件搬运工程师）

路由组件：vue-router ， 路由本质就是控制页面的跳转，就将 url 和你写的vue页面进行绑定

状态存储：vue-store ， 比如存储用户登录信息，不会因为你关闭了页面，你的登录信息就会从浏览器中消失，vue-store  会管理那些有状态的数据

axsiox： 后端api接口发送请求，  也就是 框架的api 层，真正和后端进行交互的地方

各种拦截器：可以做到拦截的地方分别是： vue-router 和 api 请求层， 比如 用户登录不允许访问某个页面 或者 当后端报错时候在页面进行友好提示
