1. 导入客户

```plain
接口： 客户导入校验
/api/lease/customer/customerExcelExport/checkCustomerExcel

数据唯一性校验 + 格式校验 + 系统是否存在校验 + 动态字段配置

接口：执行导入
/api/lease/customer/customerExcelExport/checkCustomerExcel/readCustomerExcel
实现逻辑: 直插入不更新

两个接口逻辑合并 /api/lease/customer/customerExcelExport/checkCustomerExcel/readCustomerExcelV2
```

```plain
接口: /api/lease/customer/customerExcelExport
实现逻辑: 固定表头 + 动态表单字段
```

3. 导出客户

```plain
接口: /api/lease/customer/customerExcelExport/export
新增入参: [
  {
      "filedName":"字段名字/表头",
      "filedCode":"字段标识/英文名"
  }
]

现有逻辑: 导出固定字段，导出字段用户无法控制
改造逻辑: 导出字段基于用户选择字段，表头动态，
实现逻辑: 查询固定业务字段 + 动态表单字段 + 动态表单字段值 = 列表
使用导出组件为新组件，基于基于前端动态传入进行导出
涉及动态表为: 
custom_field
custom_field_group
custom_field_table_value
custom_field_value

```

4. 导出动态选择列表

```plain
新增接口
 /api/lease/customer/customerExcelExport/exportFiledList 【导出字段选择】
返回字段:
系统动态字段列表
[
    {
        "filedName":"字段名字/表头",
        "filedCode":"字段标识/英文名",
        "isMust": "是否必须"
    }
]
用户上次过的字段
[
    {
        "filedName":"字段名字/表头",
        "filedCode":"字段标识/英文名",
    }
]

接口实现逻辑:
基于配置的动态表字段 + 固定字段【枚举】= 列表

用户每次选择后的字段 放入缓存方便下载直接使用
key: exportFiledList_userId



 
```
