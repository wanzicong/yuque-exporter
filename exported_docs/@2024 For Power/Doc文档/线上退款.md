<font style="color:#117CEE;">发起退款，退款产生的影响？</font>



1. **<font style="color:rgb(6, 6, 7);">账单的原始应收金额变为0</font>**<font style="color:rgb(6, 6, 7);">：</font>
    - <font style="color:rgb(6, 6, 7);">当退款申请被受理通过后，系统会将该账单的原始应收金额设置为0，表示这笔交易不再需要客户支付。</font>
2. **<font style="color:rgb(6, 6, 7);">优惠券退回给客户</font>**<font style="color:rgb(6, 6, 7);">：</font>
    - <font style="color:rgb(6, 6, 7);">如果交易中使用了优惠券，系统会将这些优惠券退回给客户。在web端，这些优惠券的使用明细状态会被标记为“已作废”，并注明原因是“因退款申请受理通过，优惠券使用已作废”。</font>
    - <font style="color:rgb(6, 6, 7);">优惠券会根据其当前状态回到“待使用”或“已过期”状态。</font>
3. **<font style="color:rgb(6, 6, 7);">小程序个人优惠券和企业优惠券变为待使用状态</font>**<font style="color:rgb(6, 6, 7);">：</font>
    - <font style="color:rgb(6, 6, 7);">在小程序中，个人和企业优惠券的状态会更新为“待使用”。对于已经使用的优惠券，系统会标记“因退款受理通过，优惠券已退回至待使用状态”。</font>
4. **<font style="color:rgb(6, 6, 7);">账单其他的减免或调整或坏账明细也作废</font>**<font style="color:rgb(6, 6, 7);">：</font>
    - <font style="color:rgb(6, 6, 7);">除了优惠券外，账单中其他的减免、调整或坏账明细也会被作废，以确保账单的准确性。</font>
5. **<font style="color:rgb(6, 6, 7);">实际应收金额变为0</font>**<font style="color:rgb(6, 6, 7);">：</font>
    - <font style="color:rgb(6, 6, 7);">最终，系统会将账单的实际应收金额设置为0，表示这笔交易完全退款，不再需要客户支付任何费用。</font>

财务操作流程: 

1. 用户填写退款申请单
2. 后台在退款列表（退款 tab 页面）中选择受理 （触发退订）
3. 触发退款流程
4. 受理通过发起退款流程 （手动）
5. 填写退款信息
6. 审核通过/拒绝 （影响金额的变化）



新建表， 退款申请页面，在订单里面发起退款流程



```sql
CREATE TABLE `refund_applications` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `order_id` VARCHAR(255) NOT NULL COMMENT '订单编号',
  `product_service_name` VARCHAR(255) NOT NULL COMMENT '商品/服务名称',
  `application_amount` DECIMAL(10, 2) NOT NULL COMMENT '申请金额',
  `application_reason` TEXT COMMENT '申请说明',
  `application_time` DATETIME NOT NULL COMMENT '申请时间',
  `refund_status` VARCHAR(50) NOT NULL COMMENT '退款状态',
  `merchant_processing_time` DATETIME COMMENT '商家受理时间',
  `refund_end_time` DATETIME COMMENT '退款结束时间',
  `refund_success_time` DATETIME COMMENT '退款成功时间',
  `refund_method` VARCHAR(50) COMMENT '退款方式',
  `remarks` TEXT COMMENT '备注'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```



发起申请，受理，发起退款，审批（配置都是非自动情况）

核心逻辑: 退款金额计算 ， 退款成功之后的数据影响变化
